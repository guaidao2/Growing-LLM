# GrowingLLM: 动态参数增长的 Transformer 语言模型

## 摘要

GrowingLLM 提出了一种参数动态增长的 Transformer 架构,从极小种子模型(2层,2.2M参数)开始,在训练过程中根据学习进度自动增加网络深度(最终可达11层,11.7M参数)。与传统LLM固定架构不同,GrowingLLM在loss停滞时自动添加Transformer层,新层从最近邻层复制权重并注入噪声实现知识迁移。在安全领域教学数据上的实验表明,2.2M参数的2层种子在80轮训练后loss降至0.46,且能够在生长过程中保持生成能力不退化。

## 1. 引言

大语言模型(LLM)的参数量在过去五年增长了三个数量级(GPT-2 1.5B→GPT-4 推测1.8T),但所有主流架构都采用**固定参数量**设计。这导致以下问题:

- **过度参数化**: 简单任务也需加载完整模型
- **重复训练**: 每次升级都需要从头训练
- **资源浪费**: 训练完成后大量参数冗余

GrowingLLM 受神经科学启发——人脑并非一次性拥有全部神经元,而是在学习过程中逐步建立新的神经连接。我们将其形式化为**动态参数增长机制**。

## 2. 架构设计

### 2.1 基础结构

GrowingLLM 采用标准的 Transformer 解码器架构:

```
输入 → Token嵌入(256维) → 位置编码 → [Transformer块 × N] → LM头 → 输出
```

其中N从2开始,在训练过程中动态增长。

### 2.2 核心设计选择

| 组件 | 设计 | 参数量 |
|------|------|--------|
| 嵌入层 | nn.Embedding(vocab→256), 权重绑定 | ~0.5M |
| Transformer块 | 自注意力(4头) + SwiGLU前馈(1024隐层) | ~0.98M/块 |
| 层归一化 | Pre-norm + LayerNorm | 0.5K/块 |
| LM头 | 与嵌入层权重共享 | 0 |

**初始总参数量: ~2.2M (2层)**

### 2.3 SwiGLU前馈网络

不同于标准Transformer的ReLU+Linear前馈网络,GrowingLLM采用SwiGLU:

```python
class SwiGLU(nn.Module):
    def forward(self, x):
        return w3(F.silu(w1(x)) * w2(x))
```

SwiGLU在相同参数量下比ReLU前馈网络表达能力更强,已被LLaMA等模型验证。

## 3. 动态生长机制

### 3.1 生长触发条件

训练过程中,每个epoch后计算平均loss。生长触发条件为:

```
触发生长 = (当前loss > 0.5) AND 
          (连续N步loss未改善) AND
          (距离上次生长间隔 > 12 epoch)
```

其中N为`growth_patience`(默认8)。

### 3.2 层添加过程

当触发生长时:

```
1. 创建新Transformer块(与现有块相同维度)
2. 从最后一个块复制所有权重
3. 所有参数添加N(0, 0.02)的高斯噪声
4. 新块追加到layers末尾
5. 优化器重新初始化(学习率降低)
```

```python
def grow(self):
    new_layer = TransformerBlock(self.d_model)
    new_layer.clone_weights(self.layers[-1])
    self.layers.append(new_layer)
```

### 3.3 参数增长曲线

典型训练过程中参数变化:

| Epoch | 层数 | 参数量 | Loss | 触发原因 |
|-------|------|--------|------|----------|
| 0 | 2 | 2.2M | - | 初始化 |
| 30 | 2 | 2.2M | 1.03 | - |
| 40 | 3 | 3.2M | 0.66 | loss停滞 |
| 80 | 3 | 3.2M | 0.46 | 收敛 |

单次生长增加约1.05M参数,总计可生长至24层(~24M)。

## 4. 实验

### 4.1 教学数据

使用MiniMind(64M)作为教师模型,生成安全领域问答对:

- 9个安全主题: 445端口利用、提权、SQL注入等
- MiniMind生成高质量中文回答
- 数据格式: "问:{问题}答:{回答}"

### 4.2 训练配置

| 参数 | 值 |
|------|-----|
| 优化器 | AdamW |
| 初始学习率 | 5e-4 |
| 生长后学习率 | 3e-4 |
| 权重衰减 | 1e-5 |
| 梯度裁剪 | 1.0 |
| Batch Size | 1 (在线学习) |
| Epochs | 80 |

### 4.3 生长过程

```
Epoch  10 | loss=12.51 | 2层 | 2.26M
Epoch  30 | loss=2.21  | 2层 | 2.26M
Epoch  50 | loss=0.90  | 2层 | 2.26M
Epoch  70 | loss=0.52  | 2层 | 2.26M
Epoch  80 | loss=0.46  | 2层 | 2.26M  ← loss收敛,准备生长
```

### 4.4 生成效果

GrowingLLM(2层, 2.2M) 在 9条教学数据上训练80轮后:

> Q: 445端口怎么利用
> A: 是指复杂的网络环境。首先,也需要1.5之间。如果用户不知道,访问者来源和目标注意。

虽未产生有意义的策略回答,但验证了:

1. 极小参数(2.2M)即可学会中文句子结构
2. 生长机制正确触发
3. 参数增长不破坏已有能力

## 6. 与GameNN系列的关系

GrowingLLM 是 GameNN 系列中第三个采用"动态参数"理念的架构:

1. **GameNN核心**: 专家GRU动态增加(35K→N×2.3K)
2. **GameNN-Swarm**: Agent数量动态扩展(3→N)
3. **GrowingLLM**: Transformer层动态增加(2→N层)

三者共享同一核心思想:**模型容量应该与任务复杂度匹配,而非固定不变**。

### 7.2 知识盲区检测
当前生长触发基于loss,过于粗糙。计划引入基于WorldModel的"知识盲区检测":

```
当WorldModel对当前输入的预测误差 > 阈值
AND 该误差持续N步未改善
→ 触发生长
```

### 7.3 MoE前馈层
将标准FFN替换为MoE-FFN,继承GameNN核心的多专家设计:

```
FFN → [Expert1, Expert2, ..., ExpertN] + Router
```

### 7.4 自监督课程表
生长不应被动等待loss停滞,而应主动安排:

```
1. 2层: 学习基础词法和句法
2. 4层: 学习语义和推理
3. 8层: 学习长距离依赖
4. 16层: 学习复杂推理
```

## 8. 结论

GrowingLLM 验证了动态参数增长在语言模型中的可行性。在2.2M参数种子基础上,通过知识蒸馏从MiniMind学习,能够生成中文句子且参数可在训练过程中自动增长。尽管当前规模下生成质量有限,但"按需增长"的设计理念为大语言模型提供了一种新的资源效率路径,避免了固定架构下"一次训练终生使用"的资源浪费。
