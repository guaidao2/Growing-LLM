# GameNN-Growing: 统一通用智能体框架

## 摘要

GameNN-Growing 是一个可在单个消费级 GPU (8GB) 上运行的自生长通用智能体框架。其核心创新在于 **参数动态增长机制**——从 2 层 Transformer(3.7M 参数)开始,在训练过程中遇到知识盲区时自动增加深度(2→N 层)、宽度(256→1024 维)和专家数(4→N 个 MoE 专家)。该框架整合了自生长语言模型(GrowingLLM)、统一生长调度(GrowthEngine)、多领域演进(DomainEvolver)和跨领域范式迁移(CrossDomainTransfer)四大模块,构成了 GameNN 系列的终极架构。

## 1. 设计哲学

GameNN 系列的核心思想:**
不要一开始就定死参数量,让模型自己决定什么时候长大。**

传统 LLM 的范式:
```
模型大小(7B/13B/70B) = 拍脑袋定 → 训练 → 发现不够 → 重新设计 → 重训
```

GameNN-Growing 的范式:
```
2 层(3.7M) → loss 卡住 → 自动加层 → 继续训 → 再加宽 → 再加专家 → ...
```
每一层和每一个专家都在"需要时"才被创建,且创建位置紧邻最相似已有组件从而实现知识迁移。

```
│    └── 冷却机制(深度8步/宽度15步/专家3步)                │
│                                                        │
│  DomainEvolver (多领域学习)                             │
│    ├── 顺序学习不同领域                                   │

│    ├── 注意力头重要性提取                                 │
│    ├── 领域相似度计算                                    │
│    └── 跨领域知识复用                                    │
└─────────────────────────────────────────────────────────┘
```

### 2.2 参数增长路径

| 阶段 | 层数 | 维度 | 专家 | 参数量 | 触发条件 |
|------|------|------|------|--------|----------|
| 种子 | 2 | 256 | 4 | 3.7M | - |
| 早期 | 2→8 | 256 | 4 | 15M | loss>0.3 + gap>0.5 |
| 中期 | 8→16 | 384 | 8 | 40M | 新领域学习 |

| 极限 | 32→N | 1024 | 32 | 500M+ | 分布式 + 网络分片 |

| 组件 | 起始参数 | 增长维度 | 触发条件 |
|------|---------|---------|---------|

| DomainEvolver | 0 (调度逻辑) | - | 调用learn() |

## 3. 创新点

### 3.1 自生长机制

不同于传统模型的固定架构设计,GameNN-Growing 在训练过程中实时监测两个指标:

1. **知识盲区**: `gap = 1 - max(softmax(logits))`, 衡量模型对当前输入的不确定度
2. **Loss 停滞**: 连续多轮 loss 不下降,说明当前容量已饱和

当任一指标超过阈值时,自动触发增长:

```python
gap > 0.5 + 持续上升 → 需要扩展容量
gap高 + 层数少     → 加深度 (新Transformer块)
gap高 + 层数够     → 加宽度 (扩展隐藏维度)  
gap高 + MoE模式    → 加专家 (新专家从最近邻复制)
```

### 3.2 跨领域范式迁移

每个领域学完后,提取"范式指纹"(注意力头重要性分布)。学习新领域时自动寻找最相似已学领域,迁移其注意力模式作为初始化:

```
领域A(网络安全): 注意力集中在"端口→服务→漏洞"模式
领域B(数学推理): 注意力集中在"条件→推导→结论"模式
→ 都是"序列因果链"推理, 共享注意力特征
→ 迁移后加速 B 的学习, 减少生长需求
```

### 3.3 统一生长调度

GrowthEngine 统一管理所有组件的生长决策:

```python
engine = GrowthEngine(model)
engine.step(avg_loss=1.2)  # 检测→决策→执行
# 可能触发: grow_depth(), grow_width(), grow_expert()
```

单层 TransformerBlock 的参数:

| 子模块 | 参数量 | 说明 |
|--------|--------|------|
| FlashAttn | ~600K | QKV投影 + 输出 |
| MoEFFN (4专家) | ~1.2M | 4×SwiGLU + Router |
| RMSNorm ×2 | 512 | 归一化 |
| **合计** | **~1.8M** | **每层占用** |

## 4. 与GameNN系列的关系

```
GameNN 起源 (坦克棋) ← MoE-GameNN (35K) ← GrowingLLM (3.7M→500M+)
          ↓                    ↓                    ↓
  专家GRU动态增长       MoE 多专家融合        自生长语言模型
          └────────────────┴────────────────────┘
                         GrowthEngine
                    (统一的生长调度器)
```

所有变体共享同一核心思想:**模型容量应该与任务复杂度匹配,而非固定不变。**

## 5. 实验

### 5.1 自生长训练 (R1 数据集 39K 条)

| 阶段 | 层数 | 参数量 | Loss | 累计时间 |
|------|------|--------|------|----------|
| 初始化 | 2 | 4.7M | - | - |
| 基础语言 | 2→8 | 15M | 3.97→1.9 | ~5h |

```
Q: 445端口怎么利用
A: 要确定445码的比例,计算如下:
   确定每个百分比的大小...

(注: 模型已掌握中文句法结构、Markdown格式、
    数学公式排版, 但语义理解仍需更多训练)
```

## 8. 结论

GameNN-Growing 验证了一个核心假设:**通用智能不需要一开始就拥有全部参数,而应该根据学习需求逐步生长。** 从 3.7M 参数种子开始,在消费级 GPU 上通过自生长机制可以训练出具备基础语言能力的模型(26M 参数,17 层),且生长潜力远超单卡显存限制(支持深度+宽度+专家三维生长)。
