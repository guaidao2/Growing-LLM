# GrowingLLM: 动态参数增长的通用智能体

## 摘要

GrowingLLM 提出一种可在单个笔记本电脑上持续进化的通用智能体架构。核心创新在于: **(1) 参数动态增长**——从 2 层 Transformer(1M 参数)开始,在遇到知识盲区时自动增加深度(层数)和宽度(隐藏维度),可逐步扩展至 20 层以上; **(2) 多领域演进**——通过经验回放机制在不同领域(网络安全、数学推理、代码生成等)上顺序学习而不遗忘; **(3) 自适应部署**——当模型增长超出本地算力时,自动分片卸载到网络,实现"本地轻量 + 网络深度"的透明推理。

## 1. 引言

大语言模型(LLM)的主流范式是"一次训练,终生使用":预训练一个固定参数量的模型,在特定任务上微调后部署。这种范式有三个根本矛盾:

1. **模型容量与任务复杂度不匹配**: 简单问答也需加载完整模型,复杂推理也无法动态扩展
2. **持续学习与固定架构的矛盾**: 学完一个领域后学新领域要么遗忘旧知识,要么模型不够用
3. **本地部署与模型增长的矛盾**: 模型一旦变大就只能在云端运行,失去了本地隐私和离线能力

GrowingLLM 从 GameNN 系列架构(详见 [Game-nn-O](https://github.com/guaidao2/Game-nn-O))中继承了"按需增长"的设计理念,并将其系统化为一个通用智能体框架。

## 2. 架构设计

### 2.1 整体结构

```
┌────────────────────────────────────────────────┐
│                  GrowthEngine                    │
│  (统一生长调度器: 知识盲区检测 → 决策 → 执行)    │
└──────────────┬──────────────────────────────────┘
               │ 管理
     ┌─────────┼──────────┐
     ▼         ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐
│LLM     │ │MoE-    │ │GNN     │ ← 可独立使用
│Growing │ │GameNN  │ │Encoder │
│层+宽度  │ │专家GRU  │ │节点维度 │
└────────┘ └────────┘ └────────┘
     │
     ├── DomainEvolver (多领域学习)
     └── NetworkShard (自适应卸载)
```

### 2.2 GrowingLLMv2 核心架构

```
起始: 2层, 192维, 4头, BPE词表(8K) → ~1M参数
生长路径: 
  深度:  2 → 3 → 4 → ... → N 层 (每步+~740K)
  宽度:  192 → 256 → 384 → 512 维 (每步+~1.5M)
终点: 最高 24层 × 512维 → ~40M参数
```

**改进项:**

| 旧问题 | 解决方案 |
|--------|----------|
| 字符级分词 | BPE分词器,支持中英文混合 |
| 只能加层 | 深度+宽度双维度生长 |
| 生长靠loss | 知识盲区检测(预测置信度) |
| 生成O(L²) | KV Cache增量解码 |
| 遗忘旧知识 | 经验回放(10K记忆池) |
| 本地跑不动 | NetworkShard网络分片 |

### 2.3 BPE分词器

支持中英文混合文本的 Byte-Pair Encoding:

```
输入: "扫描到445端口可以尝试MS17-010漏洞利用"
初始分词: ['扫','描','到','4','4','5','端','口','可','以','尝','试','M','S','1','7','-','0','1','0','漏','洞','利','用']
BPE合并: ['扫描','到','445','端口','可以','尝试','MS','17','-','010','漏洞','利用']
```

### 2.4 KVCache增量解码

生成时缓存每层的 Key/Value 张量,避免重复计算已生成的 token:

```python
# 首次: 全量计算
x = embed(tokens) → layers → logits

# 后续: 增量计算  
x = embed(new_token) → layers(使用KV Cache) → logits
```

复杂度从 O(L²) 降至 O(L),在 512 长度下加速约 256 倍。

## 3. 知识盲区检测

### 3.1 盲区定义

知识盲区 = 模型对当前输入的"不确定度",用预测分布评估:

```
gap = 1 - max(softmax(logits))
gap ∈ [0, 1], 越大越不确定
```

### 3.2 生长决策

```
gap > 0.6 + 持续上升 → 需要扩展容量

决策规则:
  gap高 + 层数少(<4)     → 加深度
  gap高 + 层数多(≥4)     → 加宽度  
  gap低 + loss停滞        → 加深度
  gap低 + loss持续下降     → 不生长(现有容量足够)
```

### 3.3 世界模型辅助

与 GameNN 的世界模型联动,当 WorldModel 对"执行动作后的结果"预测误差持续偏高时,也触发对应组件的生长:

```
WorldModel预测误差 > 阈值 × 持续N步
  → GameNN加专家GRU (如果现有专家不够用)
  → GNN加编码维度   (如果图结构太复杂)
  → LLM加层         (如果语言理解不足)
```

## 4. 多领域演进

### 4.1 DomainEvolver

```python
evolver = DomainEvolver(llm, growth_engine)

# 顺序学习多个领域,每学一个自动评估是否需增长
evolver.learn('网络安全',   security_texts)    # 2层 → 4层
evolver.learn('数学推理',   math_texts)        # 4层 → 6层 (新领域触发增长)
evolver.learn('代码生成',   code_texts)        # 6层 → 6层 (已有容量够用)
```

### 4.2 经验回放机制

为防止灾难性遗忘,每次学习新领域时混合回放旧领域数据:

```
训练batch = 70%新领域数据 + 30%经验回放(从10K记忆池采样)

记忆池策略: FIFO,自动淘汰最旧经验
```

### 4.3 领域级Checkpoint

每个领域学习完成后保存独立权重文件,可在推理时按需加载:

```python
models/growth/domain_网络安全.pth
models/growth/domain_数学推理.pth  
models/growth/domain_代码生成.pth
```

## 5. 自适应网络卸载

### 5.1 NetworkShard

当模型增长超出本地 GPU 显存时,自动分片卸载:

```
检测: 每步推理前检查显存使用率
触发: 使用率 > 80% + 已配置网络端点
动作: 
  1. 保留前1/3层在GPU (特征提取)
  2. 后2/3层通过网络转发到另一台机器
  3. 网络不可用时自动回退CPU

对调用方完全透明: model.reply("你好") 无需改代码
```

### 5.2 网络协议

```
请求: {tensor: [[...], ...], layer_idx: 5, total_layers: 15}
响应: {output: [[...], ...]}
格式: JSON over HTTP
```

## 6. 与GameNN系列的关系

GrowingLLM 是 GameNN 系列中的语言模型组件,与 MoE-GameNN(决策核心)和 GNNEncoder(图编码)共享统一的 GrowthEngine:

| 组件 | 参数起始 | 生长维度 | 触发条件 |
|------|---------|---------|---------|
| GrowingLLM | 1M (2层×192维) | 层数+维度 | 盲区高/新领域 |
| MoE-GameNN | 35K (3专家×64维) | 专家数 | 预测误差大 |
| GNNEncoder | 2K (64维) | 编码维度 | 图结构复杂 |

三者可独立使用,也可通过 GrowthEngine 统一调度。

## 7. 实验

### 7.1 参数增长曲线

| 阶段 | 层数 | 维度 | 参数量 | 触发原因 |
|------|------|------|--------|----------|
| 初始化 | 2 | 192 | 1.0M | - |
| 基础语言 | 2→4 | 192 | 2.5M | loss 停滞 |
| 领域扩展 | 4→6 | 192→256 | 5.2M | 新领域盲区 |
| 复杂推理 | 6→8 | 256→384 | 12.1M | 长文本理解不足 |
| 上限 | 8→12 | 384→512 | 25.8M | 多领域综合 |

### 7.2 推理速度 (KV Cache)

| 生成长度 | 无Cache | 有Cache | 加速比 |
|----------|---------|---------|--------|
| 32 | 12ms | 8ms | 1.5× |
| 128 | 48ms | 14ms | 3.4× |
| 512 | 780ms | 42ms | 18.6× |

## 8. 未来工作

- **自动课程表**: 生长不应被动等待盲区,而应主动安排学习顺序
- **多模态扩展**: 将图像/音频编码器加入 GrowthEngine 管理
- **联邦进化**: 多个 GrowingLLM 实例通过网络交换生长经验

## 9. 结论

GrowingLLM 验证了"在单个笔记本电脑上运行一个能持续进化的通用智能体"的可行性。通过 BPE 分词器、知识盲区检测、双维度生长、KV Cache、经验回放和网络卸载等机制的组合,一个起始仅 1M 参数的模型可以逐步学习网络安全、数学推理、代码生成等不同领域,并在模型增长超出本地算力时自动分片使用网络资源。

本架构的所有代码均已开源,详见 https://github.com/guaidao2/Growing-LLM
